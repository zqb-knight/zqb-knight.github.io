插入排序，合并排序，快速排序比较

姓名：赵泉斌 学号：2017211268 班级：2017211305班

一． 概述

1.1 实验要求

  • 分别设计并实现插入排序、合并排序、快速排序的算法。
  • 设计测试数据集，编写测试程序，用于测试正确性，算法复杂性，效率。

1.1 实验环境

  • 操作系统：Windows 10

  • IDE: Visual Studio 2017

二． 程序设计

​ 程序主要采用面向对象的方式实现，将快速排序和合并排序封装成类，并将测试程序封装在了类中，增加了程序的可移植性。其中插入排序较为简单，故将插入排序直接写在了主程序中。

​ 本次实验的主要目标是测试三种排序算法的时间复杂度和效率，而三种算法又有不同的版本，不说明算法形式·容易造成评测的混乱。为了方便对算法进行评测，故声明三种算法的实现形式分别为：

  • 插入排序：直接插入排序。

  • 合并排序：不做其他优化，通过递归方式实现的合并排序。

  • 快速排序：以首元素为标杆元素的快速排序。

    下面有三种算法的实现代码，均为标准的形式，可以跳过。

​ 在三种排序算法的内部又分别增加了计时模块，正确性校验模块，比较次数与腾挪次数比较模块。

三． 测试数据集设计

3.1 插入排序数据集

  • 最好情况：数据集升序 $O(n)$
  • 最坏情况：数据集降序 $O(n^2)$

3.2 合并排序数据集

  • 每种情况基本相等 $O(nlogn) $

3.3 快速排序数据集

  • 最好情况：每次调用partition时首元素刚好为中位数 $O(nlogn)$
  • 最坏情况：数据集升序和降序 $O(n^2)$

  • 分析：

    1. 在随机数据情况下，快速排序的综合速度最快。
    2. 快速排序在数据集已经排好序的情况下，性能出现了严重的退化。
    3. 插入排序在正序情况下，速度非常快。
    4. 插入排序在随机情况下，所需时间指数倍增加。
    5. 合并排序在各种情况下速度基本相似，因为每种情况均会进行递归和合并操作。
